## Setting Up Your Own MQTT Broker (Recommended for Schools)

### Why Use Your Own Broker?

**Advantages:**
- ✅ Privacy - Data stays on your network
- ✅ Reliability - No dependency on internet
- ✅ Speed - Faster local communication
- ✅ Security - Control access completely
- ✅ No limits - Unlimited messages

**Disadvantages:**
- ❌ Requires setup time
- ❌ Need dedicated device (Raspberry Pi)
- ❌ Basic network knowledge needed

---

## Option A: Raspberry Pi with Mosquitto

### Hardware Needed:
- Raspberry Pi (any model, Pi Zero W works)
- MicroSD card (8GB minimum)
- Power supply
- Ethernet cable (or WiFi)

### Installation Steps:

**1. Install Raspberry Pi OS:**
```bash
# Download Raspberry Pi Imager
# Flash Raspberry Pi OS Lite to SD card
# Boot up Raspberry Pi
```

**2. Install Mosquitto:**
```bash
sudo apt update
sudo apt install mosquitto mosquitto-clients -y
sudo systemctl enable mosquitto
sudo systemctl start mosquitto
```

**3. Configure Mosquitto:**
```bash
sudo nano /etc/mosquitto/mosquitto.conf
```

Add these lines:
```
listener 1883
allow_anonymous true

# For WebSocket support (needed for web dashboard)
listener 8884
protocol websockets
```

**4. Restart Mosquitto:**
```bash
sudo systemctl restart mosquitto
```

**5. Find Raspberry Pi IP Address:**
```bash
hostname -I
# Example output: 192.168.1.100
```

**6. Update Your Code:**

In `smart_edubuddy.ino`:
```cpp
const char* mqtt_server = "192.168.1.100"; // Your Pi's IP
const int mqtt_port = 1883;
```

In `dashboard.js` and `teacher-panel.js`:
```javascript
const MQTT_BROKER = 'ws://192.168.1.100:8884/mqtt';
```

### Testing Your Broker:

**From another computer on same network:**
```bash
# Subscribe to test topic
mosquitto_sub -h 192.168.1.100 -t test/topic

# In another terminal, publish a message
mosquitto_pub -h 192.168.1.100 -t test/topic -m "Hello"
```

## Security Considerations

### For Public Brokers:
⚠️ **Important:** Anyone can subscribe to your topics!

**Recommendations:**
- Use unique topic names with random suffix
  - Example: `edubuddy_school123_question`
- Don't send sensitive information
- Monitor for unexpected messages
- Consider encrypted communication

### For Private Brokers:
✅ **Much More Secure**

**Add Username/Password:**

Edit `/etc/mosquitto/mosquitto.conf`:
```
allow_anonymous false
password_file /etc/mosquitto/passwd
```

Create password file:
```bash
sudo mosquitto_passwd -c /etc/mosquitto/passwd teacher
# Enter password when prompted

sudo systemctl restart mosquitto
```

Update code with credentials:
```cpp
mqttClient.setUsernamePassword("teacher", "your_password");
```

## Troubleshooting MQTT Issues

### Problem: Can't Connect to Broker

**Check:**
1. Internet connection working?
2. Broker address correct?
3. Port number correct (1883 for TCP, 8884 for WebSocket)?
4. Firewall blocking connection?

**Test with MQTT tool:**
- Download: MQTT Explorer or MQTT.fx
- Try connecting to broker
- Check for error messages

### Problem: Messages Not Received

**Check:**
1. Both devices connected to same broker?
2. Topic names exactly match (case-sensitive)?
3. QoS level appropriate?
4. Check Serial Monitor for errors

**Debug:**
```cpp
// In Arduino code, add debug prints
if (client.connect(mqtt_client_id)) {
  Serial.println("Connected!");
  Serial.print("Subscribing to: ");
  Serial.println(topic_question);
}
```

### Problem: Connection Keeps Dropping

**Solutions:**
1. Increase keepalive time:
```cpp
client.setKeepAlive(60); // 60 seconds
```

2. Add reconnect logic (already in code)

3. Check WiFi signal strength

4. Use wired connection if possible

## Performance Tips

### For Best Performance:

**1. Use Local Broker:**
- Raspberry Pi on same network
- Sub-millisecond latency
- No internet dependency

**2. Quality of Service (QoS):**
```cpp
// QoS 0 = At most once (fastest, may lose messages)
client.publish(topic, message, false); // QoS 0

// QoS 1 = At least once (recommended)
client.publish(topic, message, true); // QoS 1
```

**3. Message Size:**
- Keep messages small (<1KB)
- Use short topic names
- Compress data if needed

**4. Retained Messages:**
```cpp
// Last message saved on broker
client.publish(topic, message, true); // retained = true
```
